clash轮询加tor监听多个torcc加python异步多线程实现匿名ip轮询

只暴露一个 SOCKS5 端口（比如 9050），但在后台通过多个 tor 实例 轮询出口 IP，实现负载均衡 + IP 自动切换 + 高并发处理。以下是你的目标拆解：

✅ 目标总结：

  • 只监听一个端口（比如 9050）
  • 后台多个 tor 实例（比如 9051, 9052, …）
  • 主代理程序统一监听 9050，并轮询/负载均衡转发到不同的 tor 实例
  • 每个 tor 实例独立运行，使用不同的 SocksPort 和 ControlPort
  • 支持绿色成功、红色失败的彩色输出
  • 脚本结束时清除启动的所有 tor 子进程
  • 隐藏 torcc 回显，防止混乱

✅ 解决方案：

我们需要一个主控程序，它：

  1. 启动多个 tor 实例（监听不同端口）
  2. 启动一个“中间代理”（比如基于 pysocks 或 asyncio-proxy）
  3. 监听一个统一端口（如 9050），将流量轮询分发到每个 tor 实例上
  4. 提供彩色日志
  5. 程序结束时自动清理所有进程
