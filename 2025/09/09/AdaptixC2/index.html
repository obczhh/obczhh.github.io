AdaptixC2编译到运行

AdaptixC2

Adaptix 是一个为渗透测试人员设计的可扩展的后渗透和对抗模拟框架。Adaptix 服务器使用 Golang 编写，以允许操作员的灵活性。GUI 客户端使用 C++ QT 编写，使其可以在 Linux、Windows 和 MacOS 操作系统上使用。

Features 功能特性

  • Server/Client Architecture for Multiplayer Support
    支持多人游戏的服务器/客户端架构
  • Cross-platform GUI client
    跨平台 GUI 客户端
  • Fully encrypted communications
    完全加密的通信
  • Listener and Agents as Plugin (Extender)
    监听器和代理作为插件（扩展器）
  • Client extensibility for adding new tools
    客户端可扩展性，用于添加新工具
  • Task and Jobs storage 任务和作业存储
  • Files and Process browsers
    文件和进程浏览器
  • Credentials, Targets and Screenshots Managers
    凭证、目标和屏幕截图管理器
  • Socks4 / Socks5 / Socks5 Auth support
    Socks4 / Socks5 / Socks5 认证支持
  • Local and Reverse port forwarding support
    本地和反向端口转发支持
  • BOF support BOF 支持
  • Linking Agents and Sessions Graph
    连接代理和会话图
  • Agents Health Checker 代理健康检查器
  • Agents KillDate and WorkingTime control
    代理杀毒日期和工作时间控制
  • Windows/Linux/MacOs agents support
    Windows/Linux/MacOs 代理支持
  • Remote Terminal 远程终端
  • AxScript Engine AxScript 引擎

项目管理菜单。关闭项目将断开当前项目与 C2 服务器的连接并关闭项目区域。新建项目将打开一个新的登录窗口。

Session Table 会话表格

会话表显示哪些代理已注册到 AdaptixServer。

用户字段显示所使用的访问令牌。会话表有代理标记：已终止、非活动、未链接、无响应。操作员可以将代理标记为非活动或活动，还可以选择条目的颜色或整行的颜色。将代理标记为无响应的时间在会话表设置中设置。

将鼠标悬停在睡眠字段上将显示当前的 KillDate 和工作时间设置（如果已设置）。在工作时间外，代理将被标记为“无工作时间”。

双击代理行将打开代理控制台。代理通过会话表的上下文菜单进行管理。

Sessions Graph 会话图表

没有可视化表示，很难追踪和理解由 4 个或 5 个链接代理组成的链。这时，会话图就派上用场了。

会话图以自然的方式展示了代理链。每个代理会话都有一个电脑图标，表明其操作系统和权限级别。如果图标是红色的，则表示代理正在特权进程中运行。灰色图标表示代理当前正在被检查（可能处于非活动状态）。

防火墙图标代表代理的出站点。每条线都包含负责处理代理数据的监听器的名称。虚线绿色表示使用了”外部”监听器。

一条连接一个代理会话到另一个代理的实线表示两个代理（”内部”监听器）之间的链接。这些线包含监听器名称和链接名称

代理通过会话表的上下文菜单进行管理。

Agents console 代理控制台

代理控制台看起来是这样的。每个命令包括操作员名称、任务 ID 和代理类型。

控制台会保存输入命令的历史记录，并支持命令自动补全。通过按 Tab 键，控制台支持本地路径的自动补全，而 Ctrl+Tab 将打开文件选择窗口。控制台拥有自己的菜单，包含许多设置。

Agents 代理

Beacon Configuration 配置

此版本的 Beacon 代理并非为规避 AV 或 EDR 而设计，因此不符合 OPSEC 标准。Beacon 主要旨在测试和展示 AdaptixC2 的能力。然而，Beacon 代理可以轻松扩展和修改，使其兼容多种 AV 规避方法。

Beacon 目前仅支持 Windows 操作系统。

Beacon 可以生成以下格式的 x64 和 x86 架构版本：EXE、DLL、服务可执行文件、Shellcode。

KillDate 参数设置 Beacon 的死亡日期和时间（GMT+0 格式）。

WorkingTime 参数设置 Beacon 活跃的时间间隔（目标主机的本地时间）。

  • Legit-DLL 已设置为 DLL。此选项将生成用于侧载的 DLL，其中包含指定 DLL 的所有导出项。

  • 服务名称已设置为服务可执行文件。
