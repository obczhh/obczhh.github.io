灰帽黑客操作指南来自onion(英译中)

灰帽黑客操作指南

最近，我对自己用于服务检测、渗透测试、漏洞赏金、黑客攻击、红队攻防以及随手研究的指令和步骤进行了一次梳理。我决定将它们统一记录在一个地方，并根据需要随时扩充。以下是我在进行安全研究时使用的步骤和工具清单，主要是近期常用的那些。我会不定期更新工具库，因为有些工具会因停止开发而过时，随之会有新工具取而代之。

另一个原因是我喜欢有一套操作流程可以遵循。黑客攻击是一个复杂的过程，有时会变得非常混乱，因此坚持某种指南是很有意义的，这样我才不会意外遗漏任何环节。针对所有目标的攻击，并没有唯一的金科玉律或万能指南。

虽然通用方法可能有一个大体框架，但一切都会根据你的实际发现而改变。此外，关注新的应用程序和脚本也是值得的，因为有时你可以将它们加入自己的工具库中，以加速或自动化某个步骤——当然，除非你追求的是纯手工挑战带来的名声和荣耀。

这里有一些可以发现新工具的网站：

  • KaliLinuxTutorials
  • KitPloit

怎么说呢，我并不是那种会发布 CVE 编号、挖掘零日漏洞（0-day）或发现全新漏洞的黑客。换句话说，我也许能做到，但可能要花非常长的时间，且发现后的回报与投入并不相称。由于我这人既懒散又贪婪，还认为成名是对自由的束缚，所以我更像是一个“种族歧视者”式的脚本小子。

我的黑客行为核心在于寻找那些“低垂的果实”（容易入手的目标），并利用对方技术人员犯下的错误。

从某种角度看，这正是黑客攻击的本质：我了解一个系统本该是什么样子的，而拥有该服务的人可能知道，也可能不知道。所以，这关乎于对方是否有相关知识，或者是否存在知识盲区。正是这些安全漏洞或配置不当，导致了服务的弱点暴露在外。这本质上是一场知识竞赛：要么有人利用知识来保障服务安全，要么我利用知识来接管它。

这段描述可能有点乱，但对我来说很受用。我训练得越多，发现的漏洞就越多；发现的漏洞越多，我接管的服务也就越多。

而且，这并不像电影里演的那样，你下单买个黑客服务就能无孔不入。防御严密的地方是无法攻破的，事实就是如此（或者说我还没达到那个水平）。面对防御严密的地方，唯一的突破口就是击垮它的主人——那个活生生的人。一旦人被攻破，我们就能像他一样拥有访问权限。

不过我不“黑”人，我把社会工程学技巧留着给自己谋福利，让别人按我的意愿行事，或者让他们同意一些通常不会同意的事。相比之下，我更享受操纵人的感觉。

所以，我只黑那些“能被黑”的东西，而不是“应该被黑”或“我想黑”的东西。

中间插播一张为了这篇文章专门生成的 AI 搞怪图——这可是当今每个黑客博客的标配。现在的互联网到处充斥着这种 AI 生成的图像，所以我也顺应潮流，把这玩意儿也放这儿。慢慢欣赏。

挺有意思的，AI 想象中的黑客空间（hackerspace）和现实中真实的黑客空间，完全是两码事。

回到黑客攻击本身。总的来说，这是一个既艰难又漫长的过程：运行工具、分析结果、理解结果、得出结论，然后重新启动工具、重新分析；而在工具运行并吐出更多结果供分析的间隙，往往要耗费数小时。你需要分析和审查海量数据，接着检查那些潜在的漏洞是否真实存在且可被利用，结果最后却发现它们根本无法使用。

也许之后只需点几下鼠标就能接管数据库或服务。又或者，你发现所有已知的选项都已排查完毕，所有线索都断了，只能放弃这次攻击。然后转向目标列表，选择下一个，再次重复数小时甚至数天的分析。结果可能还是颗粒无收。有时一个月能成功几次，有时一周能成几次，有时则会遭遇“失败马拉松”，连续几个月一无所获。

这可不像电影里演的那样令人兴奋。

所以，理论上的步骤看起来大概是这样的，虽然有时它们与实际操作完全不沾边。但你总得从某个地方开始：

  • 侦察/足迹探测 (Reconnaissance/Footprinting)
  • 扫描 (Scanning)
  • 获取权限 (Gaining access)
  • 维持权限 (Maintaining Access)
  • 清理痕迹 (Clearing tracks)

关于这些步骤及其背后的理论，网上有成堆的页面在详细描述，你肯定能搜到。下面是我在这些环节中常用的一些工具。我更多地侧重于**侦察（Reconnaissance）**和**扫描（Scanning）**，并根据发现的结果来获取权限、维持权限，或者直接拿走我想要的数据。有时我也会清理痕迹以增加被溯源的难度，但在大多数情况下，只要你的 OPSec（操作安全）做得够好，就算留下一片狼藉，也没人能抓到你。

由于我主要针对 Web 应用程序，目标大多是数据库或其他可以提交漏洞赏金（Bug Bounty）的漏洞，所以清单并不会涵盖所有领域。噢，别忘了，我可是个“资深脚本小子”。

记得时不时回来看看，我会更新工具的增减情况。

下面的列表只给出了基础命令示例，你总该花点时间去调整参数。尤其是那些涉及线程（threads）、时机（timing）、节流（throttling）、侵略等级（aggression level）以及其他能降低噪音（防发现）的参数。另外，别忘了在工具配置中添加一些 API Key，这样能获得更好的结果。

环境配置

我所有的操作都在专门的物理机和虚拟机上的 Kali Linux 中完成。物理机用于执行那些耗时较长的任务，它是我的主力机，（在搞渗透时）24 小时运行，其余时间关机。虚拟机则用于测试和运行一些可以快速完成的自定义工具或任务。

此外，在进行测试和渗透时，系统会连接到托管在我 VPS 上的 VPN；有时我也会使用 PIA (Private Internet Access) VPN；或者在渗透 .onion 域名的隐藏服务时，将 Tor 作为 SOCKS 代理来转发流量。

总结如下：

  • 操作系统： Kali Linux
  • 网络： VPN（个人 VPS 搭建 + Private Internet Access）

足迹探测 (Footprinting)

足迹探测仅仅是搜集目标的一般性信息，它是侦察过程的第一步。这包括被动搜集（如 DNS、公开记录、WHOIS、社交媒体）和主动搜集，即与系统产生交互（如 Ping、网络拓扑映射、Banner 抓取）。这一切都是为了了解系统的形态及其运作方式。

建议尽可能将“侦察”与“扫描”区分开来，以免侦察阶段的操作过早暴露，导致整个行动夭折。

  • WHOIS 查询： 获取域名注册信息。
  • Nslookup/Dig： 获取 DNS 相关信息。
  • Traceroute（路由跟踪）： 确定通往目的地的路径。
  • Google Dorking（谷歌黑客语法）： 利用特定的搜索指令，挖掘那些不易获取的信息。
  • 全网搜索引擎： 如 Shodan、ZoomEye、Censys、GreyNoise。
  • WAF 识别： 识别被 Web 应用防火墙隐藏的后端应用。
  • NMAP： 进行基础扫描。

通过此阶段的任务，可以搜集到以下信息：

  • 域名、IP 地址以及网段信息。
  • 网络架构相关信息。
  • 员工详情、电子邮件地址及联系方式。
  • 正在使用的操作系统及其软件版本。
  • 安全策略与配置情况。
  • WHOIS 注册数据。
  • 系统层面的工具信息。

Google Dorks（谷歌黑客语法）

使用 Fast Google Dorks Scan（https://github.com/IvanGlinkin/Fast-Google-Dorks-Scan） 进行自动化 Dork 扫描。已从 GitHub 下载并安装在 /opt 目录下。

Pagodo (https://github.com/opsdisk/pagodo)用于自动化抓取和搜索 Google Hacking 数据库（GHDB）。已从 GitHub 下载并作为 Python 虚拟环境安装在 /opt 目录下。

-i - 指定两次 Dork 搜索之间的最小延迟（秒）。不要设得太短，否则你的 IP 会很快触发 HTTP 429 错误（请求过多）。

-x - 指定两次 Dork 搜索之间的最大延迟（秒）。不要设得太长，否则搜索过程会耗费极长时间。

想尽可能快地向 Google 发送 7300 多次搜索请求是行不通的。一种解决方案是准备一组 HTTP(S)/SOCKS 代理池，并将其传递给 pagodo。

Amass

Amass 是一款网络映射工具，利用开源信息搜集和主动侦察技术来发现攻击面和外部资产。

同样，为了获得更精准的结果，值得配置各种 API。配置文件路径为：$HOME/.config/amass/config.ini。

reconftw

reconftw 旨在对目标域名执行自动化侦察，它集成了一系列最优工具集来完成扫描并发现漏洞。

执行扫描（全量扫描 + 断点续传）：

  • **-r (Recon)**：全量侦察，但不包含主动攻击。
  • **-s (Subdomains)**：子域名枚举、Web 探测及子域名接管。
  • **-p (Passive)**：仅进行被动侦察。
  • **-a (All)**：全量侦察加上主动漏洞检查。
  • **-w (Web)**：针对特定 Web 目标进行漏洞检查。

reNgine

reNgine 是一个顶级的 Web 应用侦察套件，旨在为安全专家、渗透测试人员和漏洞赏金猎人极大提升侦察效率。

我的建议： 将其安装在配有 Docker 的纯净 Ubuntu 服务器上。

WAF 检测

使用 wafw00f(https://github.com/EnableSecurity/wafw00f) 来测试目标应用程序是否隐藏在 Web 应用防火墙（WAF）之后。

Nmap 扫描

  • -sS：SYN 扫描（半开放扫描），侵入性较小。
  • -T2：将扫描速度设置为“礼貌”模式（polite），降低被 IDS/IPS（入侵检测/防御系统）发现的风险。
  • -sV：探测开放端口上的服务版本。
  • –randomize-hosts：随机化扫描主机的顺序。
  • -D RND:10：诱饵扫描（Decoy scan），通过伪造 10 个随机源 IP 来掩盖真实的扫描源。

你也可以使用其他扫描类型，例如：

  • -sA：TCP ACK 扫描，用于检查端口是否被过滤（防火墙规则测试）。更多详情可以参考官方文档。

gowitness

gowitness 是一款由 Golang 编写的网站截图工具。它利用 Chrome Headless（无头浏览器）通过命令行生成 Web 界面的截图，并提供一个方便的报告查看器来处理结果。

  • Profil3r：一款 OSINT 工具，可以查找个人在社交网络上的潜在个人资料及其电子邮件地址。已从 GitHub 安装在 /opt 目录下的 Python 虚拟环境中。https://github.com/amitrajputfff/Profil3r

扫描与枚举 (Scanning and Enumeration)

根据侦察阶段的结果（例如检测到的 CMS 类型、运行的服务等），我会运行其他扫描器，主要是漏洞扫描器、端口扫描器或全能型工具，以收集尽可能多的攻击向量。这一步是为了发现存活主机并与其交互，从而获取特定端口、运行服务以及软件版本的更多细节。

扫描完成后，我还会进行枚举（Enumeration），从已识别的服务中提取详细且具体的信息。其核心在于挖掘用户名、网络共享、Web 服务器路径、配置信息等。

ZAP (Zed Attack Proxy)

在这一步中，当我开始接触 Web 应用程序时，我会运行 ZAP。它能对网站进行爬取（Crawl），并运行被动及主动扫描。你也可以使用 Burp Suite 来完成这项任务。

Nmap 进阶扫描

参数详解：

  • -sC：使用默认脚本进行扫描。
  • -sV：探测服务版本信息。
  • -O：探测操作系统类型。
  • -oA：将结果以三种主要格式输出到指定文件名。
  • -T4：设置扫描强度为 4 级（侵略性快速扫描）。

通用漏洞扫描器

Nikto https://github.com/sullo/nikto是一款经典的 Web 服务器漏洞扫描器，用于发现潜在的问题文件、检查过时的服务器版本以及特定的服务器配置问题。

Skipfish https://www.kali.org/tools/skipfish/ 可能会有人问：为什么还在用这个？这工具已经十多年没更新了。但在处理一些使用老旧技术栈的网站时，它依然非常有效。

Wapiti https://github.com/wapiti-scanner/wapiti 基于 Python3 的 Web 漏洞扫描器。它采用“黑盒”测试方式，不读取源码，而是像 Fuzzer（模糊测试工具） 一样工作：爬取页面、提取链接和表单，然后注入 Payload 观察是否存在错误消息或异常行为。

其他专项扫描与测试

SSLScan https://github.com/rbsec/sslscan 用于测试启用 SSL/TLS 的服务，确定其支持的密码套件（Cipher Suites）及安全性。

ShellShockHunter https://github.com/MrCl0wnLab/ShellShockHunter 一个简单的工具，用于批量测试臭名昭著的 Shellshock（破壳）漏洞。

Bbot https://github.com/blacklanternsecurity/bbot 一款为黑客设计的递归式互联网扫描器，功能极其强大且模块化。

WebCopilot https://github.com/h4r5h1t/webcopilot 自动化工具，先枚举子域名，然后过滤出可能存在 XSS、SQLi、Open Redirect、LFI、SSRF 和 RCE 的参数，最后进行漏洞扫描。

作为后端，我使用 Burp Collaborator https://portswigger.net/burp/documentation/collaborator 或 InteractSH https://app.interactsh.com/。

子域名 (Subdomains)

构建子域名列表以扩大攻击范围，或者在可能的情况下尝试接管（Takeover）它们。

Subfinder https://github.com/projectdiscovery/subfinder：快速的被动子域名枚举工具。

为了获得更好的结果，建议配置 API。

Knock https://github.com/guelfoweb/knock：子域名扫描工具。已从 GitHub 安装在 /opt 目录下。

Bbot https://github.com/blacklanternsecurity/bbot：专为黑客设计的递归式互联网扫描器。已从 GitHub 安装在 /opt 目录下。

dnsReaper https://github.com/punk-security/dnsReaper：面向攻击者的子域名接管工具。已从 GitHub 安装在 /opt 目录下，并配置为 Python 虚拟环境。

Subzy https://github.com/PentestPad/subzy：子域名接管漏洞检测工具。可以先使用 Subfinder 或 Bbot 手动生成列表，然后将其作为目标添加至 Subzy 中。

模糊测试与暴力破解 (Fuzzing and Bruteforcing)

SQL 注入 (SQLi)

针对 SQL 注入，先获取一份带有参数的 URL 列表是非常值得的，你可以从 ZAP、Burp Suite 或其他爬虫工具中导出。

你也可以直接利用 SQLMap 进行爬取：

  • –batch：非交互模式，自动接受 SQLMap 的默认选项。
  • –crawl：指定爬取网站的深度。
  • –forms：解析并测试表单。

**jSQL Injection **https://github.com/ron190/jsql-injection：一款用于自动化 SQL 注入的 Java 应用程序，提供图形界面（GUI）。已下载至 /opt 目录。

**ParamSpider **https://github.com/devanshbatham/ParamSpider：从 Web 归档的隐藏角落挖掘 URL，用于漏洞猎取、模糊测试或深度探测。 虽然你可以通过 ZAP/Burp 获取带参数的 URL 或复制 POST 请求，但 ParamSpider 在提取具有价值的 URL 方面表现非常出色。

Arjun https://github.com/s0md3v/Arjun：HTTP 参数发现套件，用于寻找隐藏的参数。

最后一点建议

还有一件事：不要使用太多的“多合一”工具，或者那些标榜能帮你搞定一切的“赏金猎人助手”。现在有很多新项目试图将我上面列出的各种工具自动化组合起来，比如 scan4all 或 rekono 等。这些项目在有人维护和开发时确实很酷，但最终大多寿命很短且存在大量 Bug。

没有任何一个工具能为你包办一切，并直接生成一份能让你直接拿去换赏金的报告。所以，当你看到一个工具的依赖项里包含了 Nikto、Sublist3r、SQLMap、Gobuster 等工具时，这通常意味着它只是一个“套壳”，在后台运行这些工具并收集结果，或者在模块间传递数据。这对你并没有实质性的提升。

如果你想实现自动化，自己写一个简单的脚本：只做一件事，然后把结果传给下一环，而你最终仍然需要亲自分析结果。自动化固然很棒，但过度自动化会导致错误，让你错失细节。如果花了一整天跑扫描，最后却发现结果有误或遗漏了关键信息，那才是真的浪费时间。

这些“全能型”工具对新手的唯一作用是帮助理解流程和所使用的工具。你可以遵循它们的逻辑，复现它们的步骤，从而掌握基础。在我上面的清单中，你也发现了一些自动化工具，但我只选择了那些专注于单一任务、在某一类别中活动范围较窄的工具。
